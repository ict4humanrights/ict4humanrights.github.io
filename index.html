WebMirror

What is the WebMirror?

It is a lightweight linux application capable of periodically retrieving the content of a website under threats (censorship, hacking) and in the event of the unavailability of the website, able to host static mirrors using resilient cloud or other infrastructure.

Easy to install and configure, with a small memory footprint, it can run in the background of an old spare Linux-based PC or on a dedicated server; as long as there is a connection to the Internet, the tool can retrieve the content of the website and distribute it through its web-server.

The application is packaged with the mirroring tools and a pre-configured, lightweight, and SSL-compatible web server (nginx). Administrators will, in an intuitive fashion via command line, be able to provide the URLs of targeted websites, specify mirroring criteria and intervals, and automatically begin hosting a mirror if a given website is found to be unavailable. It can also be configured to automatically send an email to a specific address, or to post a URL to a determined Facebook page once the website is down to continue distribute the blocked content.

List of features

  • Threaded, parallel mirroring, allowing the user to simultaneously mirror as many websites as his/her computer’s processing power will allow.
  • Direct integration with the web server package nginx, which is ideal for serving high-volume static web sites.
  • Independent routine for monitoring the uptime of censored/targeted servers, which regularly checks accessibility for the target sites.
  • Automated posting of mirror URL to Facebook profile or via email alerts.
  • User-configurable delay between requests for content from the mirrored site, to keep server load manageable.
  • All web mirroring platform traffic is disguised as traffic coming from a regular web browser, to ensure that the web servers fulfill requests for content as they would for regular users.

What is the NewsObfuscator?

It is lightweight linux application that can aggregate and deliver content such as online news headlines, politically sensitive information or any other feed-based message within citizen movements using small and secure image files, whose content cannot be easily read by SMS message or web content filtering technologies.

Once installed and configured on a Linux based PC, it will automatically and at a customizable time interval, gather headlines from selected RSS feeds and output them as a series of lightweight GIF images.

The images can be automatically posted on a Facebook page or by email, to be then redistributed by the prefered channel of the citizen movement, such as MMS messaging, but also via social networks (Facebook, twitter, etc...).

The system is highly responsive to issues of security and reliability: every image is assigned a unique random name and file size. Levels of random noise to frustrate automated optical character recognition systems are imposed on every picture.

The application is packaged with a lightweight web server (nginx) capable of securely hosting the generated images while using a minimal memory footprint. This makes it easy to run in the background of a desktop PC but also of an old spare Linux based machine as long as it is connected to the Internet.

List of features

  • Generation of “micro-newspaper” images from news headlines in user-specified RSS feeds
  • Capable of handling input text from a wide variety of latin and non-latin languages (Arabic, Farsi, Chinese, Burmese for example are available)
  • Support for multiple fonts, capable of handling Unicode-encoded text in most languages
  • Images are generated with semi-random file names and file sizes, to avoid automated detection
  • User-configurable image distortion and random noise and shapes, designed for frustration of optical character recognition
  • Activation, deactivation, and configuration reloading using a command-line daemon
  • User-customizable image generation interval
  • Automated upload of micro-newspaper images to Facebook profiles, pages, and application pages, complete with newspaper names and timestamp
  • Closer integration with the nginx web server package, including automated configuration of nginx to support the platform
